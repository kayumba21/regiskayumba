Enterprise Network Design & Configuration

Full-scale enterprise network simulation in Cisco Packet Tracer — multi-floor, multi-department topology with advanced routing, security, wireless, and VoIP.

Cisco Packet Tracer OSPF HSRP ASA Firewall VLANs EtherChannel VoIP Wireless

Configuration Steps

  1. 0
    Network Design & Beautification Designed and laid out the full topology in Cisco Packet Tracer — logical placement of routers, switches, firewall, servers, wireless controllers, and end devices across three floors.
  2. 1
    Basic Settings, SSH & Standard ACL for SSH Configured hostnames, banners, password encryption, console/VTY lines, and SSH access. Applied standard ACLs to restrict SSH management access to authorized hosts only.
  3. 2
    VLAN Assignment — Access & Trunk Ports Created and named VLANs for each department (Sales, HR, Finance, LOM, ICT, R&D). Assigned access ports on Layer 2 switches and configured trunk ports between all switches.
  4. 2.1
    STP PortFast & BPDUGuard on Access Ports Enabled PortFast on all access-facing switch ports to skip STP forwarding delay, and activated BPDUGuard to protect against rogue switches being plugged into end-device ports.
  5. 3
    EtherChannel Bundled physical links between the core Layer 3 switches using EtherChannel (LACP) to increase bandwidth and provide link redundancy between distribution and access layers.
  6. 4
    Subnetting & IP Addressing Planned and implemented the full IP addressing scheme — subnetted address space for each VLAN, WAN links, management interfaces, DMZ, and loopbacks across all devices.
  7. 5
    HSRP, Inter-VLAN Routing & DHCP Helper Configured HSRP on the Layer 3 switches for gateway redundancy per VLAN. Set up inter-VLAN routing via SVIs and added ip helper-address on each SVI to forward DHCP requests to the central DHCP server.
  8. 6
    Static IPs for DMZ / Server Farm Manually assigned static IP addresses to all servers in the DMZ — DNS, Web, Email, and FTP — ensuring stable, predictable addressing for service-to-service communication and firewall ACLs.
  9. 7
    DHCP Server Configuration Configured DHCP pools on the central DHCP server — one pool per VLAN with correct subnet, gateway, and DNS settings. Excluded static ranges to prevent conflicts.
  10. 8
    OSPF on Firewall, Routers & Switches Deployed OSPF as the dynamic routing protocol across the firewall, edge routers, and Layer 3 switches. Advertised all internal networks and configured passive interfaces where appropriate.
  11. 9
    Firewall Interface Security Zones & Levels Defined security zones on the Cisco ASA (Inside, Outside, DMZ) and assigned security levels. Applied interface-level policies to enforce zone-based traffic control.
  12. 9.1
    Firewall Routing — OSPF + Static Routes Enabled OSPF on the ASA to redistribute internal routes and added static routes for DMZ and internet-bound traffic to ensure correct packet forwarding through security zones.
  13. 10
    Firewall Inspection Policy Configured modular policy framework (MPF) on the ASA — class maps, policy maps, and service policies to inspect and control traffic flows between zones including deep packet inspection for key protocols.
  14. 11
    Wireless Network Configuration Set up the Cisco Wireless LAN Controller and configured SSIDs mapped to department VLANs. Connected lightweight APs per floor and tested wireless client association and IP assignment via DHCP.
  15. 12
    VoIP Configuration Configured the Cisco Voice Gateway and CUCM, assigned directory numbers, and set up IP phones across departments. Configured voice VLANs and QoS markings for voice traffic prioritization.
  16. 13
    Verification & Testing End-to-end testing of all configurations — ping tests, traceroutes, DHCP lease verification, HSRP failover simulation, OSPF neighbor checks, SSH access validation, and wireless/VoIP connectivity confirmation.
